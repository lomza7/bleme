# Audit des agents BLEME — demandes de modification
_Audit multi-agents (10 agents : 6 auditeurs par agent + chaîne + modèles → synthèse → contrôle non-négociables). Généré le 2026-07-08._

## Résumé exécutif
DIAGNOSTIC. Le socle de BLEME est sain (repli gabarit conforme, validation humaine + hash, mentions légales figées dans les templates), mais la couche IA sous-performe de façon SILENCIEUSE. Trois pathologies transverses, toutes vérifiées dans le code : (1) CONFORMITÉ INCOHÉRENTE — le filtre anti-conseil hasAdvice() n'existe qu'en copies dupliquées dans brief.ts et escalation.ts et est ABSENT des deux chemins de courrier les plus utilisés (generateLetter dans letters.ts, generateAdaptedResponse dans replies.ts) et des alertes de Nora ; un pronostic peut donc partir dans une relance ou une mise en demeure sans garde-fou (viole le non-négociable #2). (2) MÉMOIRE ASYMÉTRIQUE — la synthèse vivante (caseMemo) circule vers les agents de rédaction/revue (Marius, Léna-replies, Jeanne-devil) mais PAS vers les agents d'extraction (Nora vision+texte, Léna-inbox), et la boucle Jeanne→Sacha→Marius est OUVERTE en phase 3 (runDevilReview écrit devil_review sans recompute → l'escalade de Marius ne voit jamais les objections de Jeanne). (3) INPUTS PAUVRES ET PROMPTS HORS-SUJET — Marius reçoit des centimes bruts et une consigne de chiffrage impossible à tenir ; Nora code en dur confiance et extrait-source, confond émetteur/débiteur (un seul champ « partie ») et lit la facture en flash-lite avec le montant attendu injecté (ancrage) ; le prompt de Sacha décrit une mission (notifications) non câblée et ignore sa seule tâche réelle (la synthèse 9 sections) ; Basile est un agent fantôme (kind « admin » rejeté).

STRATÉGIE. Trois chantiers, dans l'ordre. (A) FERMER LA CONFORMITÉ ET LES BOUCLES (P0, effort faible, impact fort) : un seul module lib/ai/guardrails.ts (ADVICE_RE + hasAdvice), appliqué sur subject ET body de TOUS les rédacteurs, au grain fin (section/point, jamais tout-ou-rien) ; un helper touchCase() = recomputeCaseProgress + refreshLivingBrief appelé après CHAQUE mutation (dont runDevilReview) ; caseMemo injecté en contexte_dossier obligatoire dans Nora et Léna-inbox via un helper agentInput(). (B) RÉÉCRIRE LES PROMPTS ET ENRICHIR LES INPUTS (P0/P1) : personas calibrées, invariants portés par le prompt et non par la consigne inline, euros pré-formatés côté serveur, faits sourcés + pièces datées + mentions_obligatoires + historique courriers passés en entrée, schémas de sortie structurés (griefs pour Léna, sections pour Sacha, statut/gravité pour Jeanne, champs riches vision pour Nora), et routage du courrier « réponse à contestation » vers Léna. (C) MODÈLES EN DEUX TIERS + MOA CHIRURGICAL (P1/P2). Point de gouvernance : le runtime réel est « hermes » partout, donc on agit sur hermes_model, PAS sur le label model (trompeur en console, à corriger pour l'honnêteté). TIER HAUT (contenu vu/envoyé, raisonnement adverse, synthèse partagée) → anthropic/claude-sonnet-5 : Marius, Léna, Jeanne (fallback hors-MOA), Sacha-brief, Basile. TIER BAS (confirmation mécanique, notifications) → anthropic/claude-haiku-4-5 : Nora-texte, Sacha-notif. Vision Nora → google/gemini-2.5-flash (au lieu de flash-lite). MOA activé UNIQUEMENT sur Jeanne (analyse contradictoire = seul cas où la diversité inter-modèles paie), agrégateur claude-opus-4.8, 2 proposeurs diversifiés (deepseek-v4-pro + kimi-k2.6, retirer hermes-4-70b redondant). MOA IMPÉRATIVEMENT OFF sur Nora : la branche MOA (client.ts:338) précède la branche hermes et ne transmet ni attachments ni modelOverride — l'activer enverrait l'OCR sans l'image.

RÉSOLUTION DES CONTRADICTIONS. Nora-texte : l'audit agent proposait de garder hermes-4-70b, la stratégie modèles propose haiku-4-5 — je tranche pour haiku-4-5 (tâche étroite de confirmation, moins cher, suffisant). Jeanne exécutant : l'audit agent gardait glm-5.2 comme proposeur ; la stratégie modèles réduit à 2 proposeurs sans glm — je tranche pour 2 proposeurs diversifiés (deepseek-v4-pro + kimi-k2.6) + agrégateur opus-4.8, fallback hors-MOA sonnet-5 (la diversité prime sur le nombre). Sacha : je retiens la scission en deux clés (sacha-brief sonnet-5 / sacha-notif haiku). Marius/Léna MOA : OFF par défaut (courriers ancrés sur gabarit + validation humaine = plancher de risque déjà tenu), option d'une clé dédiée pour la seule mise en demeure / réponse-litige transmissible. Signal de pilotage transverse : instrumenter agent_runs sur le taux de simulated=true et d'échecs de parsing par agent — c'est le meilleur détecteur d'un couple prompt/modèle trop faible, aujourd'hui invisible.
